# 开发任务（Tasks）

## M1：认证与基础用户管理

- [ ] 配置 JWT 认证（guard、middleware、异常处理）。
- [ ] 实现登录 / 注册 / 刷新 token / 登出接口。
- [ ] 在中间件中集成 RBAC：支持 `role:admin,instructor,student` 检查。
- [ ] 提供用户基础接口：当前用户信息、用户列表（仅 admin）。
- [ ] 管理员分配角色接口：给用户添加 / 移除角色。

## M2：课程与章节模块（图文）

- [ ] 设计并创建 `courses`、`course_sections`（或类似）表的迁移。
- [ ] 定义 `Course` 及章节相关模型与关系。
- [ ] 讲师 / 管理员：课程 CRUD 接口（含发布 / 下线）。
- [ ] 所有登录用户：课程列表、搜索（按标题 / 标签）、详情接口。

## M3：课程报名与学习进度

- [ ] 设计 `course_user` 报名关系表（含报名时间、状态）。
- [ ] 学员报名 / 取消报名接口，带 RBAC 控制（student）。
- [ ] “我的课程”列表接口（含简单进度信息）。
- [ ] 记录章节完成情况的接口与数据结构（后续可用于统计）。

## M4：题库与考试

- [ ] 设计题库与试卷相关表（questions、exams、exam_questions、exam_attempts）。
- [ ] 讲师：题目 CRUD 接口，支持按课程 / 标签筛选。
- [ ] 讲师：创建 / 更新 / 发布考试接口。
- [ ] 学员：参加考试、提交答案接口；自动判分逻辑（客观题）。
- [ ] 成绩查询接口（按考试 / 按学员）。

## M5：报表与优化

- [ ] 按课程统计报名人数、完成率、平均分的接口。
- [ ] 按学员统计学习情况（完成课程数、最近考试成绩）。
- [ ] 补充必要的日志、错误处理与权限校验测试。
